# Markdown 发布前检查

## 范围

发布前检查一个已有 Markdown 文件。

不要：

- 写新文章。
- 发布到平台。
- 生成图片。
- 重写主题。
- 读取无关目录。

## 输入

必填：

- Markdown 文件路径。

可选：

- 目标平台。
- 用户是否需要自动修复。

## 工作流

1. 确认文件存在。
2. 读取 Markdown 文件。
3. 检查标题、标题层级、frontmatter、链接、图片引用、代码围栏和空章节。
4. 先报告阻塞问题。
5. 如果用户要求修复，只修补目标文件。
6. 重新读取修改后的文件，并验证已修复项。

## 输出

返回：

- 阻塞问题。
- 非阻塞建议。
- 已应用修复，如有。
- 剩余风险。

## 验证

- 每条发现都引用目标文件路径。
- 代码围栏成对闭合。
- 链接和图片引用语法有效。
- 未执行任何发布动作。
