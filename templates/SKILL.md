# 示例：任务审查

## 范围

使用本 Skill 审查已有产物。

不要用本 Skill 做这些事：

- 从零创建新产物。
- 发布或上传任何内容。
- 修改用户文件，除非用户明确要求编辑。
- 读取凭据或私人账号数据。

## 输入

必填：

- 需要审查的产物或目录路径。

可选：

- 目标受众。
- 发布渠道。
- 特定风险重点。

可以安全推断时：

- 从文件扩展名或目录结构推断产物类型。
- 从周边文件推断发布准备度。

只有缺失信息会阻塞可靠审查时，才向用户提问。

## 工作流

1. 解析目标路径。
2. 只读取理解产物所需的最小文件集合。
3. 识别范围、输入、输出、副作用和发布边界。
4. 检查安全风险：密钥、私人路径、破坏性行为、隐式依赖。
5. 检查质量风险：触发条件不清、输出缺失、验证缺失、恢复策略薄弱。
6. 先写阻塞问题，再写其他内容。
7. 如果用户要求编辑，只修补解决阻塞问题所需的文件。
8. 回复前验证结果。

## 文件

可以读取：

- 目标产物。
- 周边 README 或配置文件。
- 被引用的模板或示例。

可以创建：

- 用户要求时创建审查报告。

可以修改：

- 只有明确在范围内的文件。

禁止读取：

- 凭据文件。
- 私人账号文件。
- 无关用户目录。

## 工具

只把 shell 命令用于确定性的检查：

```bash
find <target> -maxdepth 3 -type f
rg -n "token|secret|password|cookie|api_key|/Users|~/Downloads" <target>
```

不要运行破坏性命令。

## 输出

按以下顺序返回发现：

1. 阻塞问题。
2. 非阻塞问题。
3. 建议修复。
4. 已执行验证。
5. 剩余风险。

每条发现应包含：

- 严重程度。
- 文件路径。
- 证据。
- 影响说明。
- 具体修复方式。

## 验证

结束前确认：

- 目标路径存在。
- 每个阻塞问题都有证据。
- 不完整打印真实密钥。
- 建议修复仍在范围内。
- 已重新读取或检查修改过的文件。

## 失败处理

如果目标路径不存在，停止并要求用户提供有效路径。

如果必需文件无法读取，报告准确路径和原因。

如果发现疑似密钥，在回复中脱敏，并建议移除。
