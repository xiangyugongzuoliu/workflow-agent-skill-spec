# 凭据声明规范

> 本文规范 Skill 如何声明、读取和处理凭据。公开 Skill 只写占位和读取方式，不保存真实密钥。

## 1. 职责

凭据规范解决：

- 需要什么凭据。
- 凭据通过什么环境变量提供。
- 需要什么权限范围。
- 缺失凭据时怎么办。
- dry-run 是否可用。
- 如何避免泄漏。

## 2. 基本原则

- 公开仓库不保存真实凭据。
- 示例只使用占位值。
- 凭据读取方式必须明确。
- 凭据缺失时不能静默失败。
- 日志和报告不打印完整密钥。

## 3. 推荐声明

在 setup 文档或 `credentials/` 占位文件中声明：

```text
# 凭据说明

## EXAMPLE_API_KEY

用途：访问 Example API。

权限：只读。

获取方式：在服务后台创建只读 key。

配置方式：

  export EXAMPLE_API_KEY="your-key"

缺失行为：进入 dry-run；需要真实请求时停止。
```

注意：公开示例中不要出现真实 key。

## 4. 环境变量

推荐：

```json
{
  "api_key_env": "EXAMPLE_API_KEY"
}
```

不推荐：

```json
{
  "api_key": "sk-real-key"
}
```

## 5. 权限范围

凭据说明必须写清权限：

```text
只读
  可用于查询、下载、读取公开或授权数据。

写入
  可创建、修改、上传。

发布
  可公开发布内容。

管理
  可修改账号、权限、基础设施。
```

默认只申请最小权限。

## 6. dry-run

含外部写入的 Skill 必须支持 dry-run。

```json
{
  "dry_run": true
}
```

dry-run 应：

- 不发送真实写入请求。
- 不上传文件。
- 不发布内容。
- 输出将要执行的计划。
- 说明需要哪些凭据。

## 7. 缺失凭据

处理规则：

```text
凭据可选
  跳过相关检查，记录 skipped_checks。

凭据必需但支持 dry-run
  进入 dry-run，输出计划。

凭据必需且无法 dry-run
  停止，说明缺失环境变量。
```

## 8. 日志脱敏

只显示前后少量字符或完全隐藏。

```text
EXAMPLE_API_KEY=sk-...abcd
```

更安全：

```text
EXAMPLE_API_KEY=<redacted>
```

## 9. 禁止事项

- 在 `SKILL.md` 中写真实 key。
- 在示例 JSON 中写真实 key。
- 在运行日志中打印完整 key。
- 在错误消息中回显请求头。
- 默认使用高权限凭据。
- 缺凭据时伪造成功。

## 10. 检查清单

```text
声明
  [ ] 每个凭据有环境变量名。
  [ ] 每个凭据有用途说明。
  [ ] 每个凭据有权限范围。
  [ ] 缺失行为明确。

安全
  [ ] 无真实密钥。
  [ ] 日志脱敏。
  [ ] 示例使用占位值。
  [ ] 默认最小权限。

执行
  [ ] 写入操作支持 dry-run。
  [ ] 凭据缺失不会静默失败。
```
