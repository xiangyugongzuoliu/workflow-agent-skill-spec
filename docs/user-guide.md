# 用户入门规范

> 本文规范面向普通使用者的 Skill 入门说明如何编写。

## 1. 职责

用户入门文档回答：

- 这个 Skill 能做什么。
- 不能做什么。
- 最小输入是什么。
- 怎样运行。
- 输出在哪里。
- 出错时看哪里。

它面向使用者，不面向维护者。

## 2. 推荐结构

```markdown
# 使用指南

## 适用场景

## 不适用场景

## 最小输入

## 运行方式

## 输出文件

## 常见问题

## 安全说明
```

## 3. 适用场景

写具体，不写空泛价值。

推荐：

```text
当你已有一个 Markdown 文件，希望发布前检查标题层级、链接、图片引用和代码围栏时使用。
```

不推荐：

```text
帮助你高效处理内容。
```

## 4. 不适用场景

明确写排除项：

- 不从零创作。
- 不自动发布。
- 不读取私人凭据。
- 不修改范围外文件。

## 5. 最小输入

示例：

```text
必需：
  - 一个 Markdown 文件路径。

可选：
  - 目标平台。
  - 是否允许自动修复。
```

## 6. 运行方式

给自然语言示例：

```text
参考这个 Skill，检查 /path/to/article.md 是否适合发布。
```

如果需要命令，也给命令：

```bash
python scripts/python/check.py --target /path/to/article.md --output runs/example/output/report.json
```

## 7. 输出文件

说明产物位置：

```text
runs/{keyword}-{time}/
  output/
    final-report.md
```

说明报告包含什么：

- 发布判断。
- 阻塞问题。
- 非阻塞建议。
- 已执行验证。
- 剩余风险。

## 8. 常见问题

推荐格式：

```text
问题：没有生成报告。
原因：目标文件不存在或不可读。
处理：确认路径存在，并重新运行。
```

## 9. 安全说明

用户入门也要写安全边界：

- 不会默认发布。
- 不会默认删除文件。
- 不会打印完整密钥。
- 网络访问会在 setup 或 Skill 中说明。

## 10. 禁止事项

- 写成营销页。
- 只讲价值，不讲边界。
- 不写最小输入。
- 不写输出位置。
- 把维护者内部流程写给用户。
- 承诺未测试平台兼容性。

## 11. 检查清单

```text
清晰
  [ ] 能做什么明确。
  [ ] 不能做什么明确。
  [ ] 最小输入明确。
  [ ] 输出位置明确。

可执行
  [ ] 有自然语言调用示例。
  [ ] 有必要命令示例。
  [ ] 常见错误有处理方式。

安全
  [ ] 写清不会默认执行的危险动作。
  [ ] 不包含真实路径或密钥。
```
