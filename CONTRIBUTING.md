# 贡献指南

贡献应让这份规范更清晰、更安全，或更容易落地。

## 好贡献

欢迎这些贡献：

- 更清楚地解释 Skill 边界。
- 更好的模板。
- 小而真实的示例。
- 触发行为测试用例。
- 安全审查改进。
- 平台差异说明。
- 官方 Skill 行为变化后的修正。

## 不接受的贡献

不要提交：

- 真实凭据。
- 客户或委托方数据。
- 私人截图。
- 维护者专属绝对路径。
- 依赖私有基础设施的脚本。
- 营销文案。
- 没有清晰边界的“万能助手”模板。
- 无法在私人环境之外复现的示例。

## 风格

写给 Agent 和维护者看，不写营销话术。

推荐风格：

- 章节短。
- 规则具体。
- 示例可验证。
- 输入和输出明确。
- 安全边界清晰。

避免：

- 空泛质量承诺。
- 很长的概念导言。
- 多个等价选项却没有默认方案。
- 声称所有 Agent 平台都支持同一组字段。

## PR 检查清单

```text
[ ] 变更可公开。
[ ] 未加入密钥或私人路径。
[ ] 新模板有对应示例或说明。
[ ] 新脚本已记录输入和失败行为。
[ ] 新平台声明尽可能链接到官方文档。
[ ] README 链接仍指向存在的文件。
[ ] 如果变更影响发布行为，已同步更新安全和测试文档。
```

## 本地审查命令

从仓库根目录运行：

```bash
gitleaks detect --no-git --source . --redact --no-banner
rg -n "token|secret|password|cookie|api_key|apikey|client|customer|/Users|~/Downloads|private" .
find . -type f -maxdepth 3
```

打开 PR 前，逐条审查搜索命中。
